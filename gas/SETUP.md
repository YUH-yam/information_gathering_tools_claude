# GAS (Google Apps Script) セットアップ手順

時流インサイト・ログの「Googleスプレッドシート連携」「マルチ端末同期」「RSS取得プロキシ」を有効化するための手順です。所要 5〜10 分。

`Code.gs` をスプレッドシートに紐づいた GAS プロジェクトに貼り付けて、Web アプリとしてデプロイし、発行された URL をアプリの設定画面に貼り付けます。

> 注意：Google の管理画面 UI は更新されることがあります。表記が一致しないときは「Apps Script Web App デプロイ」で公式ドキュメントを最新確認してください。

---

## 0. 必要なもの

- Google アカウント（無料でOK）
- ブラウザ

無料アカウントの場合、GAS の UrlFetchApp（RSSプロキシで使用）には1日あたりの呼び出し上限があります。最新の上限値は Google の Quotas ページで要確認。

---

## 1. スプレッドシートを作成

1. Google スプレッドシートを新規作成（例：`時流インサイト・ログ_DB`）
2. シート名は何でもOK。アプリ側が自動で `articles` `insight_memos` などのシートを作ります。

---

## 2. Apps Script を開いて貼り付け

1. メニュー「拡張機能」→「Apps Script」を開く
2. 開いた `Code.gs` の中身を**すべて削除**
3. 同梱の `gas/Code.gs` の中身を**全文コピー＆ペースト**
4. 画面上部のプロジェクト名（既定で「無題のプロジェクト」）を分かりやすい名前に変更（例：`時流インサイトAPI`）
5. 「💾 保存」アイコンで保存

---

## 3. Web アプリとしてデプロイ

1. 右上「デプロイ」→「新しいデプロイ」
2. 「種類の選択」（歯車アイコン）→「ウェブアプリ」を選ぶ
3. 設定：
   - **説明**: 任意（例：v1.2）
   - **次のユーザーとして実行**: 自分
   - **アクセスできるユーザー**: 全員
4. 「デプロイ」をクリック
5. 初回は「アクセスを承認」を求められます。自分のアカウントを選び、「詳細」→「安全ではないページに移動」→「許可」と進む（自作アプリのため警告が出ますが想定通り）
6. **ウェブアプリ URL**（`https://script.google.com/macros/s/.../exec`）をコピー

---

## 4. アプリ側に URL を登録

1. 時流インサイト・ログを開き、設定画面（右上「⚙」）へ
2. 「Googleスプレッドシート連携」の「GAS Web App URL」欄に貼り付け
3. 「保存」を押す

これで完了。試しに記事を1つ保存し、「同期状況」画面のログを見て `status: ok` になっていれば成功です。

---

## 5. 動作確認

### 書き込み
- アプリで記事を保存 → スプレッドシートに `articles` シートが自動作成され、行が追加される

### マルチ端末同期
- 別の端末でアプリを開き、同じ GAS URL を設定
- 同期画面の「☁️ クラウドからプル」を押す → スプレッドシートから記事・メモが復元される

### RSS 取得（GAS 経由）
- 設定画面「RSSフィード管理」でフィードを追加 → 「📡」で取得
- アプリの「同期状況」ログに `via: gas` が記録されていれば GAS プロキシ経由

---

## 6. アップデート方法

`Code.gs` を新しいバージョンに置き換えたい場合：

1. Apps Script エディタで `Code.gs` を新しい内容で上書き保存
2. 右上「デプロイ」→「デプロイを管理」
3. 既存デプロイの「✏ 編集」→「バージョン: 新しいバージョン」→「デプロイ」
4. URL は変わらないので、アプリ側の設定は変更不要

> 「新しいデプロイ」を選ぶと URL が変わってしまうので、必ず「デプロイを管理」→「編集」を使ってください。

---

## 7. トラブルシュート

### 同期ログに `error: 401` / `403` が出る
- デプロイ設定の「アクセスできるユーザー」が「自分のみ」「ドメイン内」になっている可能性
- 「全員」に再設定して再デプロイ

### CORS エラーが出る
- アプリは `Content-Type: text/plain` で送信して CORS プリフライトを避けています
- それでも出る場合、ブラウザの拡張機能（広告ブロッカー等）が干渉している可能性

### RSS取得で `via: proxy` のままになる
- GAS URL が未保存 / 誤入力 / デプロイ失敗のいずれか
- 同期画面でログの `target_sheet: ""` の `sync_type: row` がエラーになっていないか確認

### スプレッドシートに大量の `|` 区切り文字列がある
- 配列 (tags 等) は仕様通り `|` 区切りで保存しています
- マルチ端末同期時は自動的に配列に戻ります

### UrlFetchApp の呼び出し上限に達した
- 1日のRSS取得頻度を減らす
- または、CORSプロキシ経由（設定画面）に切り替える

---

## 8. セキュリティ留意点

- **このスプレッドシートURLとGAS URLは絶対に第三者に共有しないでください**。URLを知っている人は誰でも書き込み/読み出し/外部URLフェッチが可能になります（GASの仕様）。
- 個人利用に留め、機密情報の保存には使わないでください。
- 不要になったら「デプロイを管理」→「アーカイブ」で停止できます。

---

## 9. 課金について

- このアプリの構成（Googleスプレッドシート + GAS Web App）は **無料アカウントで完結** します。
- UrlFetchApp（RSSプロキシ）には1日の呼び出し上限がありますが、個人で1日数十フィード程度なら十分間に合います。
- 上限を超える運用が必要なら、Google Workspace アカウント / Cloudflare Workers などへの移行を検討してください。
